### Task 5: Player actions (move, attack, repair)

**Files:**
- Create: `src/core/actions.gd` (`class_name Actions`)

API (all static, return `{ok: bool, events: Array, reason: String}`):

```gdscript
static func legal_moves(s: BattleState, u: BUnit) -> Array[Vector2i]  # BFS within u.move + move_bonus; blocked by units, mountains, buildings; ground units blocked by water/chasm; flying units pass over anything and may END on water (hover) but not on chasm
static func legal_targets(s: BattleState, u: BUnit) -> Array[Dictionary]  # per weapon kind; each {target: Vector2i, dir: Vector2i}
static func do_move(s: BattleState, u: BUnit, to: Vector2i) -> Dictionary
static func do_attack(s: BattleState, u: BUnit, target: Vector2i) -> Dictionary
static func do_repair(s: BattleState, u: BUnit) -> Dictionary  # action: heal 1, cap max_hp
static func resolve_weapon(s: BattleState, attacker_pos: Vector2i, weapon: WeaponDef, target: Vector2i, dmg_bonus: int) -> Array  # events; shared by mechs and vek
```

Weapon targeting rules:
- melee: 4 adjacent tiles.
- projectile: fire in 4 directions; flies until first blocker (unit, mountain, building) or edge; damages/pushes the blocker (push direction = projectile direction; pull = toward shooter). If it exits the map, no effect.
- artillery: any tile in the 4 cardinal lines at range ≥2, arcs over everything. Damage impact tile; `push_adjacent` pushes the 4 neighbors of impact outward; `splash_adjacent` damages the 4 neighbors too.
- `resolve_weapon` composes `Push.damage_tile` + `Push.displace`. Order: impact damage first, then pushes (ItB order).

### Task 6: Telegraphs + enemy AI + turn engine

**Files:**
- Create: `src/core/telegraph.gd` (`class_name Telegraph`), `src/core/enemy_ai.gd` (`class_name EnemyAI`), `src/core/turn_engine.gd` (`class_name TurnEngine`)
- Test: `tests/test_telegraph.gd`, `tests/test_turn_engine.gd`

**Telegraph model:** each vek's `intent = {weapon_id, target: Vector2i, dir: Vector2i}` where `dir` = attack direction *relative to the vek* fixed at telegraph time. Execution recomputes from the vek's CURRENT position:
- melee: hits `vek.pos + dir` (origin moves with vek — the signature redirection).
- projectile: fires from current pos along `dir`, first blocker.
- artillery: hits the absolute `target` tile regardless of vek movement (arcs are aimed at ground). Scarab keeps absolute target.
- leader_sweep: hits all 4 tiles adjacent to the boss's current pos.
- Dead vek don't attack; intent cleared on death.

**EnemyAI (deterministic, RNG-free):**
- Target scoring: prefer tiles that hit a mech (score 3) > building (2) > objective structure (4) > empty (0). Tie-break by `hash(str(s.turn) + str(vek.id) + str(candidate))` — state-derived, stable across snapshot/restore.
- Movement: BFS to best tile from which its weapon can hit the best-scored target; vek don't move onto spawn-telegraph tiles if avoidable.

**TurnEngine orchestrates the loop** (all return events the view replays):

```gdscript
static func start_battle(s: BattleState) -> Array        # initial telegraph (step 2+3 for turn 1, no attacks)
static func end_player_turn(s: BattleState) -> Array     # = enemy_attack + spawns + enemy_move_telegraph + objective check + turn++
static func check_outcome(s: BattleState) -> String      # "" | "won" | "lost_mechs" | "lost_grid" | "failed_protect"
```

`end_player_turn` order per spec: (1) vek execute intents → (2) spawn resolution (pending tiles: occupied → `spawn_blocked` + 1 dmg to blocker, retry stays pending; else `vek_spawned`) then telegraph new tiles from schedule → (3) vek move & set intents → outcome check. Survive-N: won after the enemy attack phase of turn N completes. Kill-all: won when no live vek AND no pending spawns AND empty remaining schedule. Protect: objective structure destroyed → `mission_failed`; the −2 grid power penalty is applied immediately here as events (so the view shows it); RunState just reads the final grid_power.
